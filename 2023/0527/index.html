Visualizing Neural Networks

An Intuitive Explaination of Convolutions (continued from CNNs)

A 1D-convolution is expressed: \[(f \ast g)(c) = \sum_{a+b=c} f(a) \cdot g(b)\] Let's break this down with an example. Imagine you have a ball and it is dropped multiple times. Based on symmetry, if we know the probability of the ball landing at a certain distance from where it started (denoted as x), we can also determine the probability of it starting at x distance from where it landed. Given the ball lands at "c", but we would like to know if it dropped at "a" in the previous drop. Imagine we have two probability distributions, one representing the ball's position during the first drop (denoted as f), and the other representing the probability of the ball being at a certain position if it lands at position "c" during the second drop (denoted as g). We calculate the contribution of each intermediate position by multiplying the corresponding probabilities from distributions f and g together. We then sum up these contributions from all the intermediate positions to obtain a single value representing the combined probability at the evaluated position. In simpler terms, evaluating the convolution involves looking at each intermediate position and determining how much the distributions f and g contribute to the final result. When the distributions align (while holding the g distribution still), the convolution shows a peak, indicating a higher probability of the ball being in the previous position.

This concept extends to the visual environment as well. Similarly, this can be thought of as holding one function still and sliding the other on top, multiplying and adding. In image processing, a small sized kernel is convolved across the entire image and computes a new pixel as a weighted sum of the pixels it floats over. To detect edges, we can take -1 and 1 on adjacent pixels with 0 everywhere else in the kernel (subtracting side-by-side pixels, so an edge shows when they are different).

Deciding What to Visualize

There are three components to a neural network to consider: activations, weights, and attributions. The activations correspond to each neuron and can loosely be described as what the network detected. The weights can be viewed as the compiled result of the activations, measuring how the network changes the input from one layer to the next. Attributions are not explicitly in the network, but it measures how one neuron influenced a later neuron, trying to establish a causal relationship.

One technique to establish a meaningful pattern in a network's weights is weight banding. Specifically, weight banding refers to the situation where the weights tend to cluster or form distinct bands along one dimension. This can be visually represented as sharp peaks or spikes in the weight distribution.

Considering that there are indirect interactions (i.e. residual network, the output of one neuron can pass through the additive residual stream), extended weights which are the result of multiplying adjacent weight matrices can be used. If two layers interact linearly, the expanded weights will return a true linear map, while layers which interact non-linearly return the expected value of the gradient up to a constant factor. This method which involves visualizing the weights at each spatial position and interpreting them collectively, can be particularly valuable in understanding neural networks with "bottleneck layers." Bottleneck layers are network layers designed to reduce the number of channels or dimensions, often implemented within a branch of the network. For example, by analyzing the connections and visualizing the weights in the wider layer preceding the bottleneck, we can gain insights into how the information is transformed and propagated through the network.

Circuits in Neural Networks

A neural network is composed of numerous interconnected neurons organized into layers, and the flow of information through these interconnected pathways forms circuits. Normally, these pathways indicate a set of tightly linked features and the weights between them.

Interpretability in neural networks has always been important, so researchers may want a “interpretability benchmark” in the future, though it does remain difficult to make robust statements about the behavior of holistic neural networks. Interpreting circuits may offer an advantage in this regard. They can robustly be held to a hypothesis (in visual systems, if a circuit is used to detect ears, a change in weights should change the performance of the network).
